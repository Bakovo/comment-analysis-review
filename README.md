# 评论分析复验 Skill / Comment Analysis Review Skill

> 面向社媒评论、用户评价和 AI 报告复验的双语 Codex skill。它帮助你把杂乱评论拆成可追溯、可量化、可复验的业务洞察，并在 AI 报告进入决策前拦住弱证据结论。
>
> A bilingual Codex skill for comment analysis and AI report review. It turns messy comments into evidence-backed insights and audits weak claims before they become business decisions.

## 中文版

### 这个 Skill 是做什么的

`comment-analysis-review` 是一个用于评论分析全流程的 Codex skill，适合处理社媒评论、电商评价、App Store 评价、用户反馈、竞品口碑和 AI 生成的评论分析报告。

它不只是做情绪分析或摘要，而是要求每一个可用于决策的结论都能回到：

- 原始评论证据
- 明确的分析口径
- 可复验的分子和分母
- 样本量提醒
- 方面级观点拆解
- 业务动作建议
- 置信度和下一步验证方式

### 为什么需要它

很多 AI 评论分析报告看起来很流畅，但常见问题很多：把不同分母混在一起、用极小样本做排名、把平台用户刻板印象当成证据、把不同层级的类别放在一起比较、从用户痛点直接跳到产品定位。

这个 skill 的目标是让 Codex 在做评论分析时更像一个严谨的研究助理：先拆观点，再算口径，再审证据，最后才写结论。

### 双语工作方式

这是一个 skill，但内置中英两套参考文件。Codex 会根据用户问题、原始材料和交付语言自动选择对应版本：

- 中文任务或中文报告：使用 `references/*.zh.md`
- 英文任务或英文报告：使用 `references/*.md`
- 中英混合任务：优先遵循用户要求的输出语言

为了方便 CSV、脚本和后续数据处理，结构化字段名默认保留英文，例如 `aspect`、`opinion`、`sentiment`、`evidence`、`business_action`、`confidence`。

### 适合的场景

- 社媒评论分析
- 电商和 App 评论分析
- 用户反馈归因
- 竞品口碑研究
- AI 生成报告复验
- 产品痛点和机会点挖掘
- 市场定位和传播策略校验
- 面向老板或客户的洞察报告审查

### 不适合的场景

- 自动抓取平台评论
- 替代法律、医疗、合规或品牌审核
- 用小样本直接证明市场级结论
- 在没有人工复核的情况下发布高风险结论
- 把用户评论直接当成临床、医学或监管证据

### 它会重点检查什么

- 百分比是否有明确分子、分母和范围
- 样本量是否足够支撑排序或对比
- 平台画像是否有交叉表或原文证据支持
- 不同品牌、产品、场景是否在同一层级比较
- 原始评论是否真的支持报告里的解释
- “痛点 -> 产品定位 -> 商业建议”之间是否跳跃过大
- 负面率、负面量、优先级是否被混为一谈

### Before / After 示例

弱结论：

```text

```

使用本 skill 复验后：

```text

```

弱结论：

```text

```

使用本 skill 复验后：

```text
需要改写。CPAP 是治疗设备，消费级监测产品不能直接表述为治疗替代品。更稳妥的说法是“低负担辅助监测”或“趋势记录工具”，除非另有证据支持替代治疗的说法。
```

### 推荐工作流

1. 确认数据范围：平台、时间、评论来源、筛选条件和样本量。
2. 清洗评论：去重、去广告、去空文本、去低信号模板内容。
3. 探索主题：用关键词、共现、聚类或人工分组发现候选主题。
4. 拆解观点：把一条评论拆成一个或多个 `aspect + opinion + sentiment + evidence`。
5. 量化优先级：分开看提及量、负面率、负面量、互动权重和业务影响。
6. 复验结论：检查证据、口径、样本、可比性和建议风险。
7. 输出报告：保留 caveat，把不够强的结论降级为假设。

### 输出标准

用于业务决策的结论应尽量包含：

- 分析范围
- 分子和分母
- 分母边界
- 原始评论证据
- 样本量提醒
- 指标解释：率、量、优先级不要混用
- 置信度
- 下一步验证建议

如果某个结论缺少证据、样本过小、类别混乱或策略跳跃过大，应改写、降级或删除。

## English Version

### What This Skill Does

`comment-analysis-review` is a Codex skill for end-to-end comment analysis and AI-generated report review. It is designed for social media comments, ecommerce reviews, app store reviews, customer feedback, competitor reputation research, and insight reports generated by AI.

Instead of producing loose summaries, the skill pushes Codex to connect each decision-level claim back to:

- original comment evidence
- explicit analysis scope
- reproducible numerator and denominator
- sample-size caveats
- aspect-level viewpoint extraction
- business implications
- confidence level and next validation step

### Why It Exists

Comment analysis often fails at the moment it becomes useful: turning raw feedback into business decisions. AI reports can sound confident while mixing denominators, ranking tiny samples, overclaiming platform personas, comparing mismatched categories, or jumping too quickly from user pain points to product positioning.

This skill helps Codex behave more like a rigorous research reviewer: extract viewpoints first, quantify carefully, audit evidence, and only then write conclusions.

### Bilingual Behavior

This is one skill with two language-specific reference sets. Codex should automatically choose the right version based on the user's request, source material, and expected deliverable:

- Chinese task or Chinese report: use `references/*.zh.md`
- English task or English report: use `references/*.md`
- Mixed-language task: follow the user's requested output language

Structured field names stay in English by default so CSV files, scripts, and downstream analysis remain consistent across languages.

### Best For

- Social media comment analysis
- Ecommerce and app store review analysis
- Customer feedback mining
- Competitor reputation research
- AI-generated report review
- Product pain point discovery
- Positioning and opportunity assessment
- Executive-ready insight validation

### Not For

- Scraping comments from platforms
- Replacing legal, medical, compliance, or brand review
- Treating small-sample signals as market-level conclusions
- Publishing high-risk claims without human review
- Using user comments as clinical, medical, or regulatory evidence

### What It Checks

- Whether every percentage has a clear numerator, denominator, and scope
- Whether sample size supports rankings or comparisons
- Whether platform personas are backed by cross-tab or comment evidence
- Whether brands, products, and scenarios are compared at the same level
- Whether original comments support the stated interpretation
- Whether strategy recommendations overreach beyond the evidence
- Whether negative rate, negative volume, and priority are kept separate

### Before / After

Weak claim:

```text
Fitbit has the best reputation because its positive sentiment rate is the highest.
```

Reviewed with this skill:

```text
Downgrade to hypothesis. Fitbit appears in only 3 comments, so the result is a case-level signal rather than a brand reputation ranking.
```

Weak claim:

```text
CPAP users complain about mask discomfort, so this product can be positioned as a comfortable CPAP replacement.
```

Reviewed with this skill:

```text
Rewrite. CPAP is a treatment device. A consumer monitoring product may be framed as low-burden auxiliary monitoring or trend documentation, not as a treatment replacement, unless separate evidence supports that claim.
```

### Core Workflow

1. Confirm scope: platform, time range, comment source, filters, and sample size.
2. Clean data: remove duplicates, ads, empty text, irrelevant records, and low-signal boilerplate.
3. Explore themes: use keyword counts, co-occurrence, clustering, or manual grouping to find candidate themes.
4. Extract viewpoints: split each comment into one or more `aspect + opinion + sentiment + evidence` rows.
5. Quantify priority: separate mention volume, negative rate, negative volume, engagement-weighted severity, and business impact.
6. Audit conclusions: check evidence, denominator scope, sample support, comparability, and recommendation risk.
7. Write deliverables: preserve caveats and downgrade weak conclusions to hypotheses.

### Output Standards

Decision-level insights should include:

- analysis scope
- numerator and denominator
- denominator boundary
- original comment evidence
- sample-size caveat when needed
- distinction between rate, volume, and priority
- confidence level
- recommended next validation step

Claims with missing evidence, tiny samples, category confusion, or excessive strategy leaps should be rewritten, downgraded, or removed.

### Included Scripts

These scripts are optional helpers. You can use the skill without running Python, but the scripts are useful when you have CSV outputs to validate or summarize.

Validate a viewpoint CSV:

```bash

```

```bash

```

The default priority score is a heuristic, not a universal scoring model. Adjust the weights by business context, category risk, and decision cost.
