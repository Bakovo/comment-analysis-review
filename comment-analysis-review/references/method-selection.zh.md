# 方法选择

评论分析要分层使用方法。快速方法用来找线索，决策级洞察必须回到方面级观点、原文证据和复验。

| 方法 | 适合回答 | 优势 | 风险 |
| --- | --- | --- | --- |
| 词频 | 用户最常提什么 | 快速、透明、易解释 | 不懂语义和情绪 |
| TF-IDF | 不同平台、评分、人群中哪些词更有代表性 | 适合做分组差异 | 仍是词层面 |
| 共现分析 | 哪些问题经常一起出现 | 适合发现问题链 | 噪声会放大错误关联 |
| LDA | 评论大概有哪些话题 | 经典、可解释 | 短文本和噪声场景不稳定 |
| NMF | 短评中有哪些相对清晰主题 | 电商和短评中通常更清晰 | 仍需人工命名和校验 |
| Embedding / BERTopic | 不同表达但意思相近的评论能否归到一起 | 适合社媒短文本和多样表达 | 参数和模型依赖较强 |
| 整体情感分析 | 整体口碑趋势如何 | 简单直观 | 一条评论多种情绪时会丢失细节 |
| 方面级情感分析 | 哪个方面好或不好 | 最接近业务动作 | 需要更强抽取和质检 |
| LLM 结构化标签 | 如何形成观点表和业务标签 | 灵活，适合复杂评论 | 需要一致性和幻觉检查 |

推荐主线：

```text
关键词或 TF-IDF 粗探索
-> 候选主题发现
-> 方面级观点抽取
-> 指标计算
-> 证据和复验
```

主题模型用于发现线索，不要直接把主题模型输出当作最终业务结论。
