# 复验规则

## 风险等级

审计表中的风险等级要一致：

- `high`：可能误导战略、管理层决策、产品定位、竞品排名、医疗或合规敏感表达、资源投入优先级。
- `medium`：方向可能成立，但需要补分母、限制说明、样本量提示或更窄的表达。
- `low`：主要是措辞、追溯性、格式或文档质量问题，不会明显改变结论。

## 分母规则

每个比例都必须写清楚：

- 分子
- 分母
- 分母口径
- 筛选边界

不合格写法：

```text
睡眠呼吸监测占比为 49.9%。
```

合格写法：

```text
在 12,489 条产品标注记录中，睡眠呼吸监测出现 6,230 条，占 49.9%。
```

## 排序规则

必须区分这些说法：

- 负面率最高
- 负面声量最大
- 互动加权严重度最高
- 业务优先级最高

低声量方面可能负面率最高，但不一定是业务优先级最高。

## 样本量规则

不要用极小样本做排名或市场级结论。小样本只能作为案例观察。

建议用语：

- `方向性观察`
- `案例级信号`
- `待验证假设`
- `样本不足，不能用于排名`

## 子集可复现规则

任何子集数量都要说明：

- 来源总体
- 筛选条件
- 去重逻辑
- 纳入和排除的标签
- 如果使用模型标签，说明置信度阈值

如果子集无法复现，必须降级该结论。

## 平台画像规则

不要凭平台常识推断用户画像。至少需要交叉证据：

```text
platform x label x content type x sentiment x engagement
```

如果数据只支持讨论行为，不支持人口属性，使用“平台讨论画像”，不要写成“平台真实用户画像”。

## 比较对象规则

不要把不同层级对象放在一起排名：

- 治疗方式
- 产品品类
- 品牌
- 产品线
- 医疗设备厂商
- 植入式治疗方案

先分层，再在同一层级内比较。

## 策略跳跃规则

痛点证据不等于购买意愿。购买意愿也不等于定位证明。

重点标记这些高风险跳跃：

- 治疗设备不舒服 -> 消费产品可替代治疗
- 准确性质疑 -> 已具备医疗级准确性
- 社媒讨论 -> 已验证目标人群
- 小样本情绪 -> 市场排名结论

把证据不足的策略结论改写成更窄、可验证的机会假设。

## 修正文案模式

修正弱结论时，保留有效信号，但降低结论强度：

```text
原文：Fitbit 口碑最好，因为正面率为 66.7%。
修正：Fitbit 仅出现 3 条评论，因此只能作为案例级信号，不能作为品牌口碑排名。
```

如果结论无法作为假设保留，应直接删除。
