# 方法选择

评论分析要分层使用方法。快速方法用来找线索，决策级洞察必须回到方面级观点、原文证据和抽样校验。

不要把这个文件当成绕过流程的快捷方式。方法选择必须放在 7 步流程内部：数据准备、清洗过滤、粗探索、主题发现、观点抽取、优先级统计、证据校验。

| 方法 | 怎么运行 | 适合回答 | 优势 | 注意 |
| --- | --- | --- | --- | --- |
| 词频 / 关键词 | 分词后统计词出现次数 | 用户最常提什么？ | 最快、透明、易解释 | 只看出现次数，不懂语义和情绪 |
| TF-IDF | 比较某词在某组评论中是否特别突出 | 差评、平台或人群中哪些词更有代表性？ | 适合分组差异分析 | 仍是词层面 |
| 共现分析 | 统计词或主题是否经常同时出现 | 哪些问题经常一起发生？ | 适合发现问题链和关联痛点 | 噪声会让关系图混乱 |
| LDA | 基于词袋的概率主题模型 | 评论大概可分成哪些话题？ | 经典、可解释 | 短文本和噪声场景不稳定 |
| NMF | 基于 TF-IDF 矩阵做主题分解 | 评论中有哪些相对清晰的主题？ | 电商短评中通常比 LDA 更清晰 | 仍需人工命名和校验 |
| Embedding / BERTopic | 把评论转成语义向量，再聚类并提取主题词 | 不同表达但意思相近的评论能否归到一起？ | 适合社媒短文本和多样表达 | 参数和模型依赖较强，易产生杂类 |
| 整体情感分析 | 给整条评论打正/负/中性标签 | 整体口碑如何？ | 简单直观，适合趋势监控 | 一条评论多种情绪时会丢失细节 |
| 方面级情感分析 | 拆出方面、观点、情绪和证据 | 哪个方面好/不好？ | 最接近业务决策 | 需要更强模型和抽样质检 |
| 大模型结构化抽取 | 按预设字段把评论转成结构化观点表 | 如何形成业务可用标签和洞察？ | 灵活，可输出需求、痛点、建议 | 标签一致性和幻觉风险需要控制 |

## 推荐分层

```text
词频 / TF-IDF
-> 共现分析和候选主题
-> NMF、BERTopic、Embedding 或人工归类
-> 方面级观点抽取
-> 优先级统计
-> 证据校验和业务动作
```

## 选择规则

- 需要快和透明时，用关键词。
- 需要比较平台、人群、时间段时，用 TF-IDF。
- 需要看问题链时，用共现分析。
- 评论很多且主题不清楚时，用 NMF 或 BERTopic。
- 只看趋势时，可以用整体情感。
- 输出要支持业务动作时，用方面级观点抽取。
- 最终洞察必须保留原文证据。
